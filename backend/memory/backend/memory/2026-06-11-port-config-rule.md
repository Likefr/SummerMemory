# 端口/路径/配置不许乱改 — 红线规则

**时间**：2026-06-11 23:11
**重要性**：红线（被 Likefr 明确纠正）

## 规则
- 改代码时必须保留原有端口、路径、配置值，不能擅自改
- 涉及端口变更必须先查 FRP、nginx、前端代码的对应配置
- SummerMemory 是完整开源项目，代码在 GitLab + GitHub，端口路径是固定的约定，改了就和其他环境不一致

## 教训
- 2026-06-11：分块子任务把 WS 端口从 8890 改成 11436，FRP 穿透还是 8890，导致 Graph 动画失效
- Likefr 明确说：「你不能乱篡改端口噢」「我们这个系统你可是发布了 gitlab 还有 github 的，端口都是一样的，不是你随便改的」

## SummerMemory 固定端口
- HTTP API：11435
- WebSocket：8890
- PostgreSQL：5432
